# Инструкция по чекинам коучинга

## Обзор

Модуль чекинов позволяет пользователю фиксировать состояние три раза в день (утро / день / вечер)
через бота или вручную через мини-апп. AI-коуч использует эти данные для формирования
персональных инсайтов и проактивных рекомендаций.

## Слоты чекинов

| Слот | Время (МСК) | Поля |
|------|-------------|------|
| `morning` (Утро) | 6:00–11:00 | `energy_level`, `notes` |
| `midday` (День) | 12:00–16:00 | `energy_level`, `notes` |
| `evening` (Вечер) | 17:00–23:00 | `mood`, `energy_level`, `notes`, `wins`, `blockers` |

## Модель данных

Чекин сохраняется в таблице `goal_checkins` (`db/models.py: GoalCheckin`):

```
goal_id       — опциональная привязка к цели (NULL = без цели)
user_id       — Telegram ID пользователя
energy_level  — 1..5 (1=истощён, 5=в потоке)
mood          — great | good | ok | tired | bad
notes         — рефлексия / как прошёл день (текст)
wins          — победы и достижения (текст)
blockers      — что мешало (текст)
time_slot     — morning | midday | evening | manual
check_date    — дата чекина (по умолчанию сегодня)
```

## Голосовые чекины через бота

### Как это работает

1. Пользователь отправляет голосовое (или текстовое) сообщение боту
2. Голос транскрибируется через OpenAI Whisper (`integrations/voice/stt.py`)
3. `services/voice_checkin_parser.py` классифицирует — это чекин или обычный вопрос?
4. Если чекин:
   - `detect_slot()` — определяет слот по ключевым словам / времени в тексте / текущему часу
   - `detect_date()` — определяет дату по «вчера», «3 марта», «в понедельник» и т.д.
   - `parse_checkin_fields()` — LLM-парсинг через `gpt-4o-mini` → structured JSON
   - Отображается карточка подтверждения + inline-кнопки
5. Пользователь нажимает ✅ → чекин сохраняется

### Классификатор (`is_checkin_message`)

Сначала проверяются ключевые слова (rule-based):
- Слова слотов: «утром», «вечером», «в обед» и т.д.
- Слова чекина: «энергия», «настроение», «победы», «блокеры», «прогресс» и т.д.

Если ни одно не совпало — вызывается LLM-классификатор (`gpt-4.1-nano`, 1 токен ответа).

### Определение слота (`detect_slot`)

Порядок приоритетов:
1. Числовое время в тексте (regex): «в 8 утра» → morning, «в 20:00» → evening
2. Ключевые слова слота: «утром», «вечером», «днём» и т.д.
3. Текущий час (fallback): 5–11h → morning, 12–16h → midday, 17–23h → evening

### Определение даты (`detect_date`)

- Явные алиасы: «сегодня», «вчера», «позавчера»
- Dateparser (поддерживает русский): «3 марта», «в понедельник», «15.03»
- Fallback: `date.today()`

### Карточка подтверждения

После парсинга бот показывает карточку:
```
📋 Чекин — 🌙 Вечер, сегодня

⚡️ Энергия: 😊 4/5
💭 Настроение: 😊 Хорошее
📝 День: Продуктивный день, закрыл несколько задач
🏆 Победы: Выпустил новую фичу
🚧 Блокеры: Было много переключений

Всё верно? Сохраняю чекин.
[✅ Сохранить] [✏️ Изменить] [❌ Отмена]
```

При нажатии **✏️ Изменить** бот ждёт текстовую/голосовую правку, re-парсит и показывает
обновлённую карточку.

## FSM состояния (bot/states.py)

```python
class VoiceCheckinFlow(StatesGroup):
    waiting_confirmation = State()  # ждём: сохранить / изменить / отмена
    waiting_edit         = State()  # ждём текстовую/голосовую правку
```

## Callback-данные клавиатуры

```
vci_save:{slot}:{YYYY-MM-DD}  — сохранить чекин
vci_edit:{slot}:{YYYY-MM-DD}  — перейти в режим правки
vci_cancel                     — отмена
```

Дневной чекин, энергия 2, устал после митингов, нужен перерыв

Вечерний чекин: настроение хорошее, энергия 3. День прошёл продуктивно.
Победы — выпустил фичу. Мешало — много переключений

Вчера вечером энергия была 2, устал и не выспался
