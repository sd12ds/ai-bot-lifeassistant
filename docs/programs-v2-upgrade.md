# Апгрейд программ тренировок v2 — Привязка к календарю и редактирование

## 1. Текущее состояние (проблемы)

**Модель `WorkoutProgramDay`:** хранит только `day_number`, `day_name` (текст), `template_id` (обычно null).

**Проблемы:**
- AI генерирует дни с описанием в текстовом поле `day_name`, но **не создаёт шаблоны** с упражнениями → при старте дня — пустая тренировка
- Дни **пронумерованы** (1,2,3...) **без привязки к дням недели** (Пн/Вт/Ср)
- **Нет интеграции с календарём** — тренировки не отображаются в CalendarPage
- **Нет редактирования** — нельзя менять упражнения, дни, порядок
- **Примитивная ротация** `done_count % total_days` — не знает про дни отдыха, дни недели

## 2. Целевое состояние

### 2.1 Модель данных (изменения в БД)

**`WorkoutProgramDay` — добавить поле:**
- `weekday: int | null` — день недели (0=Пн, 1=Вт, ... 6=Вс). Если null — день не привязан к конкретному дню недели.

**AI-генерация** — при создании программы LLM возвращает:
- Для каждого дня: `weekday`, список упражнений (exercise_id, sets, reps, weight_kg)
- Бэкенд **автоматически создаёт Template** для каждого дня и привязывает `template_id`

**Интеграция с календарём** — при активации программы:
- Бэкенд создаёт **событие в таблице `tasks`** для каждой тренировки:
  - `event_type = 'workout'`
  - `title = day_name` (например "Грудь + Трицепс")
  - `start_at` / `end_at` — расставляются по `weekday` на `duration_weeks` вперёд
  - `tags = ['fitness', 'program:{program_id}']`
  - `is_all_day = true` (или с конкретным временем если пользователь укажет)
- При деактивации/удалении программы → удаляются связанные события

### 2.2 Next Workout логика (улучшение)

Текущая: `done_count % total_days` — не учитывает дни недели.

Новая:
1. Найти активную программу → получить дни с `weekday`
2. Определить, какой сегодня день недели
3. Найти **ближайший будущий день** программы (сегодня или следующий по weekday)
4. Проверить, была ли уже проведена тренировка на сегодня
5. Вернуть этот день как `next_workout`

### 2.3 Редактор программы (фронтенд)

Новая страница `/fitness/program/:id` или bottom-sheet поверх ProgramsPage:

**Возможности:**
- Просмотр недельного расписания (7 колонок: Пн-Вс)
- Перетаскивание дней между слотами (или тап → выбор нового дня)
- Клик по дню → открывается список упражнений из привязанного шаблона
- Редактирование упражнений в дне: добавить, удалить, изменить sets/reps/weight
- Добавить новый день тренировки
- Удалить день (сделать днём отдыха)
- Переименовать день
- При сохранении изменений → обновляется шаблон + пересоздаются события в календаре

### 2.4 Календарь — отображение тренировок

**EventCard** — добавить тип `workout`:
- Иконка: 🏋️
- Цвет полоски: оранжевый (#fb923c)
- По клику → переход на `/fitness/workout` (с подгрузкой шаблона дня)
- Если тренировка выполнена (есть session за этот день) → отображается как done

**CalendarPage** — не меняется, т.к. уже загружает все `tasks` из API. Нужно только:
- Бэкенд: при `view=calendar` отдавать также `event_type='workout'`
- EventCard: обработать новый тип

### 2.5 AI-генерация v2

**Обновлённый промпт:**
LLM возвращает JSON с:
```
{
  "name": "...",
  "description": "...",
  "days": [
    {
      "weekday": 0,
      "day_name": "Грудь + Трицепс",
      "exercises": [
        { "exercise_id": 1, "sets": 4, "reps": 8, "weight_kg": 80, "rest_sec": 90 },
        ...
      ]
    }
  ]
}
```

**Бэкенд:**
1. Для каждого дня создаёт `WorkoutTemplate` с упражнениями
2. Создаёт `WorkoutProgramDay` с `weekday` и `template_id`
3. При активации — генерирует события в `tasks` на N недель вперёд

## 3. Порядок реализации

### Спринт 1: Модель данных + AI-генерация с шаблонами
- Миграция: добавить `weekday` в `workout_program_days`
- Обновить AI-генерацию: LLM возвращает упражнения → бэкенд создаёт шаблоны
- Обновить `next_workout` логику с учётом weekday

### Спринт 2: Интеграция с календарём
- При активации программы → создавать events в tasks
- EventCard: поддержка `event_type='workout'`
- Клик по workout event → переход на ActiveWorkout

### Спринт 3: Редактор программы
- Новая страница/модал редактирования
- Недельный вид с 7 слотами
- Редактирование упражнений в каждом дне
- Сохранение → обновление шаблонов + пересоздание calendar events

## 4. API изменения (сводка)

**Новые/изменённые эндпоинты:**
- `PUT /fitness/programs/:id` — обновление программы (name, description, days)
- `PUT /fitness/programs/:id/days/:dayId` — обновление дня (weekday, day_name, exercises)
- `POST /fitness/programs/:id/days` — добавить день
- `DELETE /fitness/programs/:id/days/:dayId` — удалить день
- `POST /fitness/programs/:id/sync-calendar` — пересоздать события в календаре

**Изменения в существующих:**
- `POST /fitness/programs/generate` — возвращает программу с шаблонами и weekday
- `GET /fitness/programs/next-workout` — учитывает weekday
- `PUT /fitness/programs/:id/activate` — создаёт calendar events
- `DELETE /fitness/programs/:id` — удаляет связанные calendar events
