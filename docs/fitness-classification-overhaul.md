# Расширение фитнес-классификации и маршрутизации

> Дата: 2026-03-18
> Статус: Реализовано

## 1. Проблема

### 1.1 Описание бага
Бот не вызывал `activity_log` при сообщении «сегодня сделал бег на дистанцию 5 км», а follow-up «за 30 минут» уходил в coaching вместо fitness.

### 1.2 Root Cause — 3 бага
1. **Fitness-агент не вызывает tool** — промпт содержал «спроси подробности»
2. **Короткий follow-up уходит в чужой домен** — «за 30 минут» не матчится ни одним маркером, sticky теряется при рестарте
3. **Потеря контекста** — каждый агент имеет свой checkpoint

## 2. Архитектура classify_intent

```
L0a  Draft Guard         — активный черновик → агент черновика
L0b  Confirmation Guard  — ожидание подтверждения
L1   force_agent         — принудительный агент из handler'а
L2   Rule-based          — classify_by_rules() по ключевым словам
L3   Sticky domain       — active_domain с таймаутом 8 мин
  └─ L3a  Short follow-up guard (НОВОЕ) — ≤5 слов + нет strong-маркеров
  └─ L3b  Full signal check
L4   LLM-классификатор   — gpt-4.1-nano
```

## 3. Изменения

### 3.1 Расширение fitness intent markers — `bot/core/intent_classifier.py`

**_FITNESS_STRONG** — полностью переписан:
- Кардио/бег: `бег `, `бегал`, `пробежал`, `пробежк`, `пробежа`
- Кардио/общее: `кардио`, `hiit`, `табата`, `интервальн`
- Кардио/аппараты: `эллипс`, `эллипсоид`, `скакалк`, `гребл`
- Плавание: `плавал`, `плаван`, `проплыл`
- Велосипед: `велосипед`, `вело `, `велотрен`
- Силовые: `жим`, `присед`, `тяга`, `подтягиван`, `отжиман`, `пресс `, `становая` и др.
- Снаряды: `штанг`, `гантел`, `тренажёр`, `тренажер`, `гриф` и др.
- Общая тренировка: `тренировк`, `тренировался`, `силовая`, `качалк`, `фитнес`
- Восстановление: `растяжк`, `стретчинг`, `йога`, `йогу`, `йогой`, `мобилити`
- Шаги/ходьба: `шагов`, `прогулк`, `прошёл `, `прошел `, `походил`

**_FITNESS_NORMAL** — переписан:
- Место/контекст: `спорт`, `зал`, `мышц` и др.
- Метрики: `подход`, `повторен`, `км `, `дистанц`, `темп `, `пульс`
- Разминка/заминка: `разминк`, `заминк`

**has_strong_signal()** — новая функция для short follow-up guard.

### 3.2 Short follow-up guard — `agents/supervisor.py`

В `classify_intent()` добавлен guard L3a:
- Если sticky активен + сообщение ≤5 слов + нет strong-маркеров другого домена → sticky без LLM
- Универсален для всех доменов

### 3.3 Усиление промпта — `agents/personal/fitness_agent.py`

Блок «2) Кардио» → «2) Кардио и активность»:
- Маппинг 12 activity types (вариант B)
- 12 few-shot примеров tool call
- 2 follow-up примера
- Усиленные правила логирования («НЕМЕДЛЕННО activity_log»)

### 4.2 Маппинг activity_type (вариант B — расширенный набор)

> **Решение:** оставляем конкретные типы (yoga, hiit, stretching и т.д.) вместо маппинга в `other`.
> Обоснование: БД `String(30)` без enum/валидации, miniapp не рендерит ActivityLog, аналитика не группирует по типу.

| activity_type | Триггерные слова |

| `run` | бег, пробежка, пробежал, бегал, джоггинг |

| `elliptical` | эллипс, эллипсоид |

Синхронизация: docstring в `tools/fitness_tools.py`, комментарий в `db/models.py` и `api/routers/fitness.py` обновлены под 12 типов. Emoji mapping в tool расширен.

## 6. Ограничения

- **In-memory sticky** — теряется при рестарте бота
- **Пробелы в маркерах** — `бег ` не сработает в конце сообщения; компенсируется формами `бегал`, `пробежал`
- **LLM-недетерминированность** — даже с few-shot LLM может проигнорировать tool call
